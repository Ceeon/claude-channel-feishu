## 关键设计决策

### 1. 三参数启动方案

不走 marketplace 注册（会被官方覆盖），而是用三个 CLI 参数组合：

```bash
claude \
  --mcp-config mcp.json \                                    # 注册 MCP server
  --dangerously-load-development-channels server:feishu \     # 绕过 tengu_harbor_ledger 白名单
  --plugin-dir /path/to/plugin                                # 加载 skills
```

这是目前第三方 channel 插件唯一可行的本地开发方案。

### 2. 状态目录与代码分离

- 代码在项目目录（git tracked）
- 运行时状态在 `~/.claude/channels/feishu/`（.env、access.json、approved/、inbox/）
- 通过 `FEISHU_STATE_DIR` 环境变量可自定义位置

### 3. .gitignore 策略

- 排除：node_modules、bun.lock、.env、access.json、approved/、inbox/
- 保留：.claude/log/（开发文档有参考价值）
- 不排除 .mcp.json（是插件配置的一部分，用 `${CLAUDE_PLUGIN_ROOT}` 变量）

### 4. README 用英文

开源项目面向全球开发者，README 用英文。开发日志（.claude/log/）保留中文，给自己看。
