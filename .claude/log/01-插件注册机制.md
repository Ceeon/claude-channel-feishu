# 01 - Claude Code 插件注册机制

## 结论

Claude Code 插件系统有严格的注册链路，缺任何一环都不会加载。

## 正确的注册链路（5 步全部必须）

### 1. Marketplace 目录结构

```
~/.claude/plugins/marketplaces/{marketplace-name}/
├── .claude-plugin/
│   └── marketplace.json       ← 插件索引，列出所有可用插件
├── external_plugins/
│   └── {plugin-name}/         ← 实际插件目录
│       ├── .claude-plugin/
│       │   └── plugin.json    ← 插件元数据（name, version, description）
│       ├── .mcp.json          ← MCP 服务器配置
│       ├── server.ts          ← 服务器代码
│       ├── package.json
│       ├── skills/            ← 插件技能（/plugin:skill-name）
│       └── node_modules/
└── plugins/                   ← 内置插件（可选）
```

### 2. marketplace.json 索引

插件必须在 marketplace.json 的 `plugins` 数组里声明：

```json
{
  "name": "feishu",
  "description": "...",
  "category": "productivity",
  "source": "./external_plugins/feishu"
}
```

### 3. settings.json 启用

```json
{
  "enabledPlugins": {
    "feishu@claude-plugins-official": true
  }
}
```

### 4. installed_plugins.json 安装记录

Claude Code 自动在 `~/.claude/plugins/cache/` 创建副本并记录：

```json
{
  "feishu@claude-plugins-official": [{
    "scope": "user",
    "installPath": "~/.claude/plugins/cache/claude-plugins-official/feishu/0.0.1",
    "version": "0.0.1"
  }]
}
```

### 5. `--channels` 启动参数

**这是最关键的一步，之前一直卡在这里。**

Claude Code 不会自动监听所有 channel 插件。必须在启动时通过 `--channels` 显式指定：

```bash
claude --channels plugin:feishu@claude-plugins-official
```

多个 channel 用逗号分隔：

```bash
claude --channels plugin:telegram@claude-plugins-official,plugin:feishu@claude-plugins-official
```

## 踩坑记录

### ❌ 尝试 1：extraKnownMarketplaces + source: "directory"

把 `source: "directory"` 指向插件目录本身。结果：`CLAUDE_PLUGIN_ROOT` 未解析，MCP 启动失败。

**原因**：directory source 应该指向 marketplace 级别目录，不是单个插件。且此 source 类型可能不被完整支持。

### ❌ 尝试 2：用户级 MCP server（~/.claude.json mcpServers）

通过 `claude mcp add --scope user feishu` 注册。MCP 服务器能启动、WebSocket 能连上，但：
- 不是插件，没有 `plugin:` 前缀
- 不绑定项目
- 没有 skills（/feishu:access 不可用）
- 不是 channel（不会推送 `<channel>` 通知）

### ❌ 尝试 3：项目级 .mcp.json

在项目目录放 `.mcp.json` + `enabledMcpjsonServers`。同上，不是插件，不走 channel 通道。

### ❌ 尝试 4：挂到官方 marketplace

把插件 symlink 到 `~/.claude/plugins/marketplaces/claude-plugins-official/external_plugins/feishu`，在 marketplace.json 里加一条。

**结果**：Claude Code 启动时会从 GitHub 重新拉 marketplace.json，手动加的条目被覆盖。且 `tengu_harbor_ledger`（channel 白名单）从服务端刷新，feishu 不在官方白名单中。

### ✅ 最终方案：--mcp-config + --dangerously-load-development-channels + --plugin-dir

三个参数组合：

| 参数 | 作用 |
|------|------|
| `--mcp-config mcp.json` | 注册 feishu MCP server |
| `--dangerously-load-development-channels server:feishu` | 绕过官方白名单，启用 channel |
| `--plugin-dir /path/to/plugin` | 加载 skills（/feishu:access 等） |

```bash
alias ccfeishu="FEISHU_STATE_DIR=~/.claude/channels/feishu \
  claude \
  --mcp-config ~/.claude/channels/feishu/mcp.json \
  --dangerously-load-development-channels server:feishu \
  --plugin-dir /path/to/claude-channel-feishu"
```

## 本地 Channel 插件开发的标准流程

```bash
# 1. 准备 MCP 配置（指定 server 启动命令）
cat > ~/.claude/channels/feishu/mcp.json << 'EOF'
{
  "mcpServers": {
    "feishu": {
      "command": "bun",
      "args": ["run", "--cwd", "/path/to/plugin", "--shell=bun", "--silent", "start"]
    }
  }
}
EOF

# 2. 准备凭据
cat > ~/.claude/channels/feishu/.env << 'EOF'
FEISHU_APP_ID=cli_xxxxx
FEISHU_APP_SECRET=xxxxx
EOF

# 3. 启动（三件套）
claude \
  --mcp-config ~/.claude/channels/feishu/mcp.json \
  --dangerously-load-development-channels server:feishu \
  --plugin-dir /path/to/plugin
```

### 6. Channel 白名单（`tengu_harbor_ledger`）

`~/.claude.json` → `cachedGrowthBookFeatures.tengu_harbor_ledger` 是 channel 插件的审批白名单。即使插件已安装、已启用、`--channels` 也指定了，如果不在这个白名单里，仍然会报 `not on the approved channels allowlist`。

```json
{
  "cachedGrowthBookFeatures": {
    "tengu_harbor_ledger": [
      {"marketplace": "claude-plugins-official", "plugin": "telegram"},
      {"marketplace": "claude-plugins-official", "plugin": "feishu"}  // ← 手动加
    ]
  }
}
```

**注意**：这个值可能被服务端覆盖（它是 cached growth book feature）。如果某天突然不生效了，重新加一次。

---

**注意**：marketplace.json 会在 marketplace 更新时被覆盖。长期方案应该用独立 marketplace 或推到 GitHub。
